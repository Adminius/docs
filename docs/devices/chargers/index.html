Wallboxen

Zur Steuerung der Ladung muss evcc mit einer Wallbox kommunizieren.

In der Konfigurationsdatei gibt es hierzu den folgenden Bereich:

charger:
  - name: wallbox
    type: ...

Auf dieser Seite findest du die Konfigurationen für alle derzeit von evcc direkt unterstützten Wallboxen. Nachdem du das Code-Beispiel für deine Wallbox angepasst und in die evcc.yaml übernommen hast, kannst du die Verbindung mit dem Befehl evcc charger testen.

$ ./evcc charger

Power:          0W
Current L1..L3: 0A 0A 0A
Charge status:  A
Enabled:        false

KEBA Connect​

Diese Konfiguration unterstützt die Wallboxen KEBA KeContact P20/P30 und BMW Wallboxen.

Vorbereitung der Wallbox​

Es muss eine sogenannte UDP Funktion über den DIP Schalter 1.3 eingeschaltet (ON) werden. Die Installationsanleitung der Wallbox hilft hier weiter.

note

Die rfid: tag: und serial sind Optional.

rfid: tag: ermöglicht es evcc den Lademodus zu starten, selbst wenn die Wallbox gesperrt ist. Zusätzlich kann man die RFID Karte einem Fahrzeug zuordnen und damit evcc sagen welches Fahrzeug angschlossen ist.

serial ermöglicht es evcc auch in einer Docker Installation mit der Wallbox zu kommunizieren. Der UDP-Port 7090 muss dafür in der Docker Konfiguration hinzugefügt werden.

chargers:
  - name: my_charger
    type: keba
    uri: 192.0.2.2
    rfid:
      tag: 765765348 # RFID tag, see `evcc charger` to show tag

Wallbe Eco/Pro​

Vorbereitung der Wallbox​

Um evcc zu erlauben das Laden zu unterbrochen und wieder zu starten, muss im Gerät der DIP Schalter 10 auf ON gesetzt sein:

Die Wallbox muss über ein Netzwerkkabel an das lokale Netzwerk angeschlossen sein. Werksseitig voreingestellt ist zunächst die IP-Adresse 192.168.0.8. Das Passwort für die Weboberfläche des Controllers lautet wallbe7213. Über die Weboberfläche können dann die Netzwerkeinstellungen an das lokale Netz angepasst werden und bei Bedarf auch weitere Grundeinstellungen vorgenommen werden.

note

Für die Steuerung mit evcc ist es wichtig dass die in den Boxen genutzten Controller eine aktuelle Wallbe-Firmware installiert haben. Dies ermöglicht sowohl die Steuerung der Stromvorgabe in feinen 0,1A-Schritten als auch die problemlose Anbindung von unterschiedlichsten Modbus-Zählern an den Controller. Ältere Firmwareversionen werden von evcc nicht mehr unterstützt und können insbesondere im Bereich der Ladestromstromvorgabe zu falschen Vorgabewerten durch evcc führen.

Die aktuelle Firmware-Datei passend zur Box ist jeweils vom Wallbe-Support kostenfrei auf Anfrage zu beziehen. Ebenfalls ist eine Schritt-für-Schritt-Anleitung erhältlich zur Durchführung des Firmwareupdates. Eine direkte Downloadmöglichkeit der jeweils aktuellen Firmware wird von Wallbe derzeit leider noch nicht angeboten. https://www.wallbe.de/firmware-updates/

Die Firmware für Wallbe Eco 2.0(s) SL-01.04.21 und Wallbe Pro BE-03.01.04 wurde mit evcc erfolgreich getestet (Stand 12/2021).

warning

Die Wallbox muss vom Strom getrennt sein, bevor sie geöffnet wird. Das Öffnen einer Wallbox sollte grundsätzlich nur durch Personen mit einer entsprechenden elektrotechnischen Qualitfikation erfolgen.

chargers:
  - name: my_charger
    type: wallbe
    uri: 192.168.0.8:502 # TCP ModBus address
    meter: # only if a charge meter is connected to the controller
      power: true
      energy: true
      currents: true

Schaltbare Steckdosen​

Die von evcc unterstützten schaltbaren Steckdosen und SmartHome Schalter, können in zwei Modi betrieben werden:

  • Charger Modus (Standard)
  • Static-Switch Modus

Der Charger Modus erlaubt die Nutzung der schaltbaren Steckdose als Ladesteuerung von Akku betriebenen Geräten (E-Motorrad,E-Bikes ...). Dabei wird der Ladestrom gemessen und von evcc angezeigt, sowie beim Erreichen des Erhaltungsladungsstroms ( < standbypower) des Akkus der Status geladen ausgegeben.

Demgegenüber dient der Static-Switch Modus zum einfachen Schalten von Verbrauchern, wie Wärmepumpen, Klimaanlagen etc. Der von der Steckdose gelieferte Status hängt dabei direkt vom Schaltzustand der Steckdose ab (An=Lade,Aus=Geladen).

note

Um im Static-Switch Modus eine aussagefähige Leistungsinformation anzuzeigen, empfiehlt es sich ein separates Custom-Meter zu definieren.

Bei allen schaltbaren Steckdosen wir der Modus über den Parameter standbypower eingestellt.

Ist der Wert größer oder gleich 0, ist der Charger Modus aktiviert. Der eingestellte Wert dient als Schwellwert in Watt. Solange die durch die Steckdose gemessene Leistung sich über diesem Schellwert befindet, wird der Ladevorgang gemeldet, fällt die Leistung unter diesen Schwellwert, ist der Erhaltungsladebereich bei vollem Akku erreicht und es wird der Ladevorgang als abgeschlossen gemeldet.

Stetzt man standbypower auf einen negativen Wert arbeitet die schaltbare Steckdose im Static-Switch Modus.

chargers:

EEBUS (experimental)​

  • Starte evcc eebus-cert (falls noch nicht geschehen)
  • Füge die Ausgabe von evcc eebus-cert in die evcc.yaml Konfigurationsdatei ein.
  • Öffne die Web Oberfläche der Wallbox, um die SKI-Gerätekennung (Identifcation Number) zu finden.
    Beim Porsche Mobile Charger Connect kann man die Information im Menü "Connections", Untermenü "Energy Manager" finden.
  • Füge den Charger hinzu
    chargers:
      - name: my_charger
        type: eebus
        ski: 1234-5678-90ab-cdef-1234-5678-90ab-cdef-1234-5678 # Enter the SKI of the wallbox which can usually be found in the wallbox web interface
        forcePVLimits: true # use Overload Protection to limit PV charging, if false PV surplus is sent as recommended charging levels to the EV, but this is currently unreliable
  • Starte evcc
  • Auf der Web Oberfläche der Wallbox, üblicherweise auf der gleichen Seite wie zuvor, sollte nun evcc erscheinen inklusive einer Option, um die Wallbox mit evcc zu verbinden (pairen).
    Bitte dies durchführen.
  • evcc sollte nun mit der Wallbox kommunizieren können.

Phoenix Contact​

Phoenix Ladesteuerung. Unterstützt Wallboxen mit EM-CP-PP-ETH, EV-CC-???-ETH und EV-CC-???-SER Controllern.

Geräte mit einer Netzwerkverbindung (-ETH) erfordern den DIP Schalter 10 auf ON. Damit ist eine Ansteuerung über ModBus möglich. Näheres findet man im Handbuch des Controllers.

  • EM-ETH (Modbus TCP)
  • EV-ETH (Modbus TCP)
  • EV-SER (Modbus RTU)
chargers:
  - name: my_charger
    type: phoenix-em-eth
    uri: 192.168.0.8:502
    meter: # only if a charge meter is connected to the controller
      power: true
      energy: true
      currents: true
chargers:
  - name: my_charger
    type: phoenix-ev-eth
    uri: 192.168.0.8:502
    meter: # only if a charge meter is connected to the controller
      power: true
      energy: true
      currents: true
chargers:
  - name: my_charger
    type: phoenix-ev-ser
    device: /dev/ttyUSB0
    baudrate: 9600 # configurable (S2/DIP 1)
    comset: 8N1
    id: 1 # configurable (S2/DIP 2–6)
